# ===================================================================
# Formation LangChain 100% Locale - requirements.txt
# Stack: LangChain 1.x + Ollama + HuggingFace + ChromaDB + Streamlit
# ===================================================================
# Strategie de pinning DEUX-NIVEAUX (standard industrie) :
#
#   NIVEAU 1 - requirements.txt (CE FICHIER) : ranges semantiques
#   --------------------------------------------------------------
#   On declare l'INTENTION : bornes basses pour empecher les regressions
#   d'API, bornes hautes (majeures) pour bloquer les v2 cassantes.
#   pip resoud LA bonne combinaison de versions => evite les conflits
#   entre paquets (ex: langchain 1.0.0 exige langgraph<1.1.0, donc
#   on ne peut pas pinner a la main les deux planchers).
#
#   NIVEAU 2 - requirements.lock.txt : versions exactes
#   --------------------------------------------------------------
#   Genere par : bash infrastructure/lock.sh
#   Contient TOUTES les versions exactes (deps directes + transitives :
#   numpy, torch, transformers, etc.). C'est CE fichier qui garantit
#   la reproductibilite parfaite entre apprenants.
#   Le Dockerfile l'utilise en priorite si present.
#
# WORKFLOW pour mettre a jour la stack :
#   1. Ajuster les ranges ci-dessous (et eventuellement bumper une borne)
#   2. Supprimer requirements.lock.txt
#   3. docker compose up -d --build  (pip resoud + installe)
#   4. Verifier que tout marche (postCreateCommand, smoke tests)
#   5. bash infrastructure/lock.sh  (capture le nouveau lock)
#   6. git commit requirements.txt + requirements.lock.txt
# ===================================================================
